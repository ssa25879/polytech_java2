## 수업 내용

### 클래스 형과 참조

- 가상세계라고 함은 보통 컴퓨터의 메모리 영역을 말함
- 인스턴스는 heap 영역 안에 확보된 메모리

> 메모리 영역
>
> | Code        | DATA      | BSS         | HEAP           | STACK |
> |-------------|-----------|-------------|----------------|-------|
> | 함수, 제어문, 상수 | 초기화된 전역변수 | 초기화 안된 전역변수 | 동적할당, malloc() | 지역변수  |
>

### 메모리 참조

>
> ```java
> Hero hero1 = new Hero(); // 클래스를 인스턴스화 -> Heap 메모리에 new 키워드로 인스턴스 생성하고, Stack 영역 hero1에 주소 저장
> hero1.hp =100;
> Hero hero2 = hero1;     // Stack 영역의 hero1 주소값을 hero2에 저장
> hero2.hp =200;
> ```

hero2.hp를 출력하면 200이 나옴
> hero2는 hero1에 저장되어 있는 Hero()의 주소값을 받아오기 때문

### Java 기본형, 참조형

- 기본형
    - int, double, float, boolean, char 등
    - 기본값이 0, 0.0, false 등
    - 키워드임 Ctrl + B 를 눌러도 이동하지 않음
- 참조형
    - String, Class, Array 등
    - 기본값이 null값임
    - 명명할 때 대문자로 (ex. String, Hero)
    - Ctrl + B 를 이용하여 참조 위치로 이동할 수 있음

## 테스트를 통한 품질 향상

### 테스트 방법

1. 수동테스트 : 사람이 직접 하는 테스트
2. 단위테스트 : TestCode - 함수를 테스트
3. 통합테스트 : 관련된 클래스나 함수를 함께 테스트

### 테스트 방법론

- 화이트박스 테스트
    - 내부 구조와 동작에 중점
    - 코드의 내부 로직, 제어 흐름, 데이터 흐름을 이해하고 검증
    - 케이스 설계시 코드의 특정 부분을 직접 확인
    - 주요 기법 : 구문 검사, 경로 검사, 조건/분기 검사
- 블랙박스 테스트
    - 내부 구조를 무시고하고 기능을 테스트
    - 내부 정보를 알 필요 없이 사용자 관점에서 테스트
    - 입력 값과 예상 출력 값에 기반하여 설계
    - 요구사항 충족여부 확인, 기능적 비기능적 요구사항을 테스트
    - 주요 기법 : 등가 분할, 경계 값 분석, 상태 전이 테스트

### 단위 테스트

- 특정 모듈이 의도한 대로 작동하는가 테스트하는 가장 작은 단위의 테스트
- 꼭 필요한 경우
    - DB : 스키마 변경, 모델 클래스 변경
    - Network : 예측한 데이터가 들어오는지 확인
    - 데이터 검증 : 예측한 데이터가 제대로 처리되는지 확인

### 수동 테스트

- main 메서드에서 테스트코드를 작성하여 진행

### 테스트케이스

- 가능한 모든것을 상정하여 테스트
    - 동등 분할
        - 유효한 범위중 하나, 범위 미만/초과값으로 테스트
    - 경계값 분석
        - 상한 경계 / 하한 경계로 테스트

### 환경 준비

- Class 이름에서 Alt + Enter -> Create Test
- 또는 동일 위치에 이름Test로 생성
- Given -> When -> Then 기법을 사용
- JUnit의 assert 함수를 활용하여 결과 검증

- 테스트 중심으로 개발하는 방법론을 TDD (Test Driven Development) 라고 함

### 예외 발생 테스트

- Exception 검증
    - assertThrows<Exception> {}
- 정상 동작 검증
    - assertDoesNotThrow {}

### 장점

1. 신속한 피드백 가능
2. 조기 장애 감지 가능
3. 더 안전한 코드 리팩터링
4. 기술적 문제를 최소화해 개발속도 안정

### 과제 중 알게 된 점

- assert 종류
    - assertEquals(정답, 비교할 값); - 두 값이 같은지 비교
    - assertNotEquals(틀려야 할 값, 비교할 값); - 두 값이 다른지 비교
    - assertSame(답, 비교값); - 두 객체가 같은 메모리 주소인지
    - assertNotSame(답, 비교값); - 두 객체가 서로 다른 메모리 주소인지
    - assertTrue(조건); - 조건이 참인지
    - assertFalse(조건); - 조건이 거짓인지
    - assertNull(객체); - 객체가 null 인지
    - assertNotNull(객체); - 객체가 null 이 아닌지
    - assertThrows(발생할 예외 타입, () -> {실행 코드}) - 특정 예외가 발생하는지 확인
    - assertDoesNotThrow(() -> {실행 코드}) - 예외가 발생하지않는지 확인
    - assertAll(검증식1, 검증식2 ...) - 여러 개의 assert를 묶어서 실행한 후 실패 내역을 한번에 출력
  > **assertAll() 검증 예시**
  > ```java
  > Cleric cleric = new Cleric("클레릭");
  > assertAll("Cleric 초기 스탯 확인",
  >     () -> assertEquals("클레릭", cleric.name),
  >     () -> assertEquals(Cleric.maxHP, cleric.hp),
  >     () -> assertEquals(cleric.maxMP, cleric.mp)
  > );
  > ```
- 람다식
    - 이름이 없는 함수를 만드는 문법
    - 필요한 순간에서 즉석으로 함수를 만들어 전달하기 위해 사용함
    - (매개변수) -> {실행할 코드} 로 작성
- 예외 타입 정리
    1. NullPointerException (NPE)
        - 참조형 변수가 가리키는 객체가 없는데 호출하거나 값을 꺼낼때 발생
    2. IllegalArgumentException
        - 메서드에 전달된 값이 논리적으로 맞지 않을 때 발생
    3. IndexOutBoundsException / ArrayIndexOutOfBoundsException
        - 배열이나 리스트의 허용 범위를 벗어난 곳에 접근하려 할 때 발생
    4. IllegalStateException
        - 객체의 상태가 메서드를 호출하기에 적절하지 않을 때 발생
- 생성자 체이닝 (Constructor Chaining)
    - 클래스에서 중복이 발생할 때 사용
    - `this()` 는 클래스 내에 있는 또 다른 생성자를 호출
    - 생성자의 첫 줄에만 `this()`를 적을 수 있음
  >   ```java
  >   // 가장 파라미터가 많은 경우를 가장 먼저 작성
  >  Cleric(String name, int hp, int mp) {
  >\    this.name = name;
  >\    this.hp = hp;
  >\    this.mp = mp;
  >   }
  >   // mp를 입력받지 않으면 maxMP로
  >   Cleric(String name, int hp) {
  >   this(name, hp, maxMP);
  >   }
  >   // 아래는 에러를 발생시키는 코드
  >   Cleric(String name) {
  >   System.out.println("이건 안됨");
  >   this(name, maxHP, maxMP);
  >   }
  > 
  >  ```
