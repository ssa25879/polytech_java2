# 2026-06-22 추상클래스와 인터페이스

## 추상클래스

### 추상클래스란

- 상속의 재료로 사용 되는 클래스
- 상세 부분이 일부 미정인 클래스
- 인스턴스화가 금지되어있음
    - `(추상클래스) name = new (추상클래스)()` 가 불가능

### 추상메서드(abstract method)

- 추상클래스 아래에 있을 수 있는 메서드
- 반드시 추상클래스 아래이어야만 가질 수 있음
- 상속받은 클래스는 추상메서드를 반드시 오버라이드 해야함
    - 단, 자식 클래스도 추상클래스라면 미룰 수 있음.

## 인터페이스

### 인터페이스의 조건

- 모든 메서드는 기본적으로 추상메서드(`public abstract`)를 가져야한다.
    - `default` 메서드, `static` 메서드 `private` 메서드를 가질 수 있다
- 필드는 가지지 않는다
    - 단, `public static final` 상수의 경우에는 가질 수 있다.

### 인터페이스 구현

- `public class (클래스명) implements (인터페이스) {}` 로 구현 가능
- `public class (클래스명) implements (인터페이스), (다른 인터페이스) {}` 로 한 클래스에서 여러 인터페이스를 구현할 수 있음

### 인터페이스에서 상수 선언

- 인터페이스에서 `double PI = 3.14` 라고 선언하면 `public static final` 이 생략이 된 것과 같음

### 인터페이스 효과

- 같은 인터페이스를 구현한 클래스들은 공통 메서드를 구현하도록 강제
- 인터페이스를 구현중인 클래스는 적어도 인터페이스에 정의된 메서드를 가지고있음이 보증

### 인터페이스간의 상속 가능

- 인터페이스는 다른 인터페이스를 상속 받을 수 있음.

### 올바른 인터페이스 구현 : 계약 조건

- Can do 원칙 (형용사형)

| 특징    | 클래스 상속           | 인터페이스 구현             |
|-------|------------------|----------------------|
| 관계 정의 | "Is-a" (차는 탈것이다) | "Can-do" (차는 탈 수 있다) |
| 다중 구현 | 불가능(단일 상속)       | 가능 (여러 인터페이스 구현)     |
| 유연성   | 부모 클래스에 종속됨      | 독립적 / 모듈화에 유리        |

### implements 와 extends의 사용방법

| 부모        | 자식        | keyword    | 다중 가능 여부 |
|-----------|-----------|------------|----------|
| class     | class     | extends    | 불가능      |
| interface | class     | implements | 가능(구현)   |
| interface | interface | extends    | 가능(상속)   |

### 상속과 구현을 동시에 사용

- `public class (클래스명) extends (상속받는 클래스) implements (인터페이스)` 로 동시에 사용이 가능함

### 추상클래스와 인터페이스 선택 기준

- 공통적인 필드와 구현이 필요하면 추상클래스
- 공통 메서드 구현을 강제하고 싶다면 인터페이스
- 다중 상속이 필요하면 인터페이스

## 정리

### 상속 재료를 작성하는 개발자의 입장/역할

- 다른 사람이 상속의 재료로 사용할 부모클래스를 만드는 입장의 개발자도 존재함
- 미래의 개발자가 효율 좋게 안심하고 이용할 수 있는 상속재료를 작성해야함
- 그것을 위해 Java 에서는 추상클래스 (AbstractClass)와 인터페이스(Interface)라는 도구를 지원함

### 추상클래스

- 내용이 정의되지 않고, 상세 정의가 정해지지 않은 메서드 앞에 abstract를 붙여서 추상메서드로 함
- 1개라도 추상메서드를 가진 클래스는 앞에 abstract를 붙여 추상클래스로 만들어야한다.
- 추상클래스는 인스턴스화 할 수 없다
- 상속의 재료로서 부모클래스를 추상클래스로 개발하면, 원치 않은 인스턴스화나 오버라이드 미구현 같은 걱정이 없어짐

### 인터페이스

- 기본적으로 추상메서드만 가지고 있는 추상클래스를 인터페이스로 특별 취급을 한다
- 인터페이스에 선언된 메서드는 자동적으로 public abstract 가 되고, 필드는 public static final이 된다
- 복수의 인터페이스를 부모로 두는 다중상속이 가능하다
- 인터페이스를 부모로 둘 경우 정의에 implements를 사용한다

### 구현체 테스트

- 추상 클래스와 인터페이스는 직접 객체 생성이 불가능
- 따라서 해당 클래스, 인터페이스를 상속하거나 구현한 구현체를 통해 테스트를 함
- 현재 코드에서는 `BookTest.java`에서 `TangibleAsset.java`와 해당 추상클래스에서 상속받은 `Asset` 추상클래스, `Thing` 인터페이스의 기능과 규칙을 검증함
