## Object 클래스

### 기본 기능

- 모든 클래스는 Object 클래스의 메서드를 가짐
- Object 타입 변수에는 모든 인스턴스 대입 가능
    - 대표 메서드
        - toString() : 문자열 표현을 얻음
        - equals() : 같은지 비교
        - hashCode() : 해시코드 값을 얻음

### toString()

- 오버라이드를 통해 내가 원하는 문자열을 출력할 수 있도록 사용할 수 있음
- `alt + insert` 를 통해 오버라이드 하여 직접 치는 것 보다 편하게 조정할 수 있음

### equals()

- 기본적으로는 주소를 확인하는 목적일 때 사용함
    - 오버라이드를 통해 동등성 규칙을 재정의 할 수 있음
- List에서 동등성 비교시 사용

### hashCode()

- 오버라이드를 통해 동등성 규칙을 정의할 수 있음
    - 소수인 31을 이용하여 hashCode()의 정확성을 높일 수 있음

#### equals() 와 hashCode()는 항상 함께 동등성 규칙을 정의해야함

### Set, Map 계열에서 equals() 보다 hashCode()를 사용하는 이유

- hashCode()는 equals() 보다 비용이 쌈
    - List는 순차 검색 구조를 가져 느린 편
- 모든 객체는 해시값을 가짐
- 동일 객체는 항상 같은 해시값을 가짐
    - 같은 해시값임에도 항상 동일한 객체인것은 아님

### Collections.sort()

- List<T> 를 받아 컬렉션 내부를 정렬시킴
- 정렬 대상은 Comparable 인터페이스를 구현해야 함
- `public static <T extends Comparable<? super T>> void sort(List<T> list)`로 선언되어 있음
    - `Comparable<T>` 는 `compareTo(T o)` 를 제공

### Comparator 인터페이스

- Comparable 인터페이스를 구현하지 않은 객체도 즉석에서 정렬 규칙을 정할 수 있음
- 오름차순, 내림차순을 내가 직접 정할 수 있음
- 작성 방법
    - `Collections.sort(정렬할 객체, new Comparator<Integer>() { (compare 방식) });`
        - `Collections.sort(정렬할 객체, (a, b) -> (compare 방식));` 로 람다식으로 표현 할 수도 있음
    - 이 때 `new Comparator<Integer>() { (compare 방식) }` 부분은 익명 클래스임
- 규칙은 `comparaTo()` 의 규칙과 동일
    - `compareTo(a, b)` 일 때
        - a > b -> 양수 (+1)를 리턴
        - a = b -> 0을 리턴
        - a < b -> 음수 (-1)를 리턴

### 얕은 복사

- 객체 자체는 새로 생성하지만, 객체 내 참조 타입 필드는 원본과 동일한 객체를 참조
    - 즉, 주소만 복사함
- 원본 객체의 참조 타입 필드를 변경할 경우 복사본의 필드도 함께 변경됨

### 깊은 복사

- 객체 자체만이 아니라, 객체 내의 모든 참조 타입 필드까지 완전히 새로 생성하여 복사함
- 원본 객체와 복사된 개체는 독립적이므로, 참조 타입 필드를 변경하더라도 복사본에서는 변경되지 않음
